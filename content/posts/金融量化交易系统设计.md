# 金融量化交易系统设计：从 tick 数据到策略回测

量化交易是工程能力、数学建模与市场直觉的交汇点。一只优秀的量化基金背后是一整套精密的工程系统——从交易所的 tick 数据落盘，到策略研究员在 Notebook 里回测，再到实盘下单、风控与清算。

这篇文章会从零开始，设计一个完整的量化交易系统，重点覆盖架构、数据、回测、风控四个核心环节。

## 一、量化交易系统的分层架构

一个生产级量化系统通常分为五层：

```
┌─────────────────────────────────────────┐
│  策略层    各种 alpha 策略、套利策略     │
├─────────────────────────────────────────┤
│  回测层    事件驱动回测引擎              │
├─────────────────────────────────────────┤
│  执行层    下单、撤单、仓位管理          │
├─────────────────────────────────────────┤
│  数据层    tick 数据、因子库、特征存储   │
├─────────────────────────────────────────┤
│  基础设施  消息总线、监控、风控          │
└─────────────────────────────────────────┘
```

每一层都要做到"无状态、可水平扩展、可独立部署"。这是支撑高频、中频、低频多种策略在同一套基础设施上运行的前提。

## 二、数据层：tick 数据的工程

### 2.1 数据类型

| 数据 | 频率 | 用途 |
|------|------|------|
| Tick / L2 订单簿 | 微秒级 | 高频做市、套利 |
| K 线（OHLCV） | 秒/分钟/小时 | 中低频策略 |
| 逐笔成交 | 毫秒级 | VWAP / TWAP 执行 |
| 基本面 | 季度 | 多因子选股 |
| 新闻 / 舆情 | 不定 | 事件驱动策略 |

### 2.2 存储选型

- **Kdb+**：华尔街默认选项，列式存储 + q 语言，单机可处理 TB 级 tick 数据
- **ClickHouse**：开源替代，OLAP 性能优秀，SQL 友好
- **Parquet on S3**：归档数据，成本最低
- **TimescaleDB**：基于 PostgreSQL，适合中小规模

## 三、特征工程：从原始数据到因子

### 3.1 因子分类

- **量价因子**：MOM（动量）、ROC（变动率）、MACD、RSI、布林带
- **波动率因子**：HV（历史波动率）、IV（隐含波动率）、Garman-Klass 估计
- **微观结构因子**：买卖价差、订单簿不平衡（OBI）、Kyle λ
- **基本面因子**：PE / PB / ROE / 资产周转率
- **另类因子**：卫星图、舆情、供应链数据

### 3.2 因子计算

用 [pandas-ta](https://github.com/twopirllc/pandas-ta) 或 [vectorbt](https://github.com/polakowo/vectorbt) 向量化计算：

```python
import pandas as pd
import numpy as np

def mom(close: pd.Series, window: int = 20) -> pd.Series:
    return close.pct_change(window)

def obi(l2: pd.DataFrame) -> pd.Series:
    """订单簿不平衡：bid_vol - ask_vol / (bid_vol + ask_vol)"""
    bid_vol = l2[l2.side == 'bid'].groupby('ts').size.sum()
    ask_vol = l2[l2.side == 'ask'].groupby('ts').size.sum()
    return (bid_vol - ask_vol) / (bid_vol + ask_vol)

def kyle_lambda(returns: pd.Series, volume: pd.Series) -> float:
    """Kyle λ：衡量单位成交量的价格冲击"""
    slope, _ = np.polyfit(volume, returns, 1)
    return slope
```

### 3.3 因子库管理

每个因子应记录：

- 计算公式
- 数据依赖（哪些表、哪些列）
- 回看窗口（lookback window）
- 更新频率
- 历史表现（IC、IR）

推荐用 [Factor Zoo](https://github.com/alphaM-factor-research/FactorZoo) 这类框架管理。

## 四、策略层：从想法到代码

### 4.1 均值回归策略示例

经典 Bollinger Band 均值回归：

```python
class BollingerReversion(Strategy):
    params = {
        'window': 20,
        'std_dev': 2.0,
        'stop_loss': 0.03,
    }

    def on_bar(self, bar):
        ts = self.close_series[-self.p.window:]
        mean = np.mean(ts)
        std = np.std(ts)
        upper = mean + self.p.std_dev * std
        lower = mean - self.p.std_dev * std

        if self.position == 0:
            if bar.close < lower:
                self.buy(size=self.risk_size(bar))
        elif self.position > 0:
            if bar.close > mean or bar.close < self.entry_price * (1 - self.p.stop_loss):
                self.sell(size=abs(self.position))
```

### 4.2 趋势跟踪策略

海龟交易法则的简化版：

```python
class TurtleBreakout(Strategy):
    params = {
        'entry_window': 20,  # Donchian 通道
        'exit_window': 10,
        'atr_window': 20,
        'risk_pct': 0.01,
    }

    def on_bar(self, bar):
        high = max(self.high_series[-self.p.entry_window:])
        low = min(self.low_series[-self.p.entry_window:])
        atr = self.calc_atr(self.p.atr_window)
        unit = (self.cash * self.p.risk_pct) / atr

        if bar.close > high and self.position == 0:
            self.buy(size=unit, stop=bar.close - 2 * atr)
        elif bar.close < min(self.low_series[-self.p.exit_window:]):
            self.close_all()
```

### 4.3 高频做市策略

订单簿不平衡做市：

```python
class OBI_MarketMaking(HFTStrategy):
    params = {
        'obi_threshold': 0.3,
        'spread_bps': 5,
        'order_size': 0.1,
        'max_position': 1.0,
    }

    def on_tick(self, tick):
        obi = self.calc_obi(tick.l2)
        mid = (tick.bid + tick.ask) / 2
        spread = mid * self.p.spread_bps / 10000

        if abs(self.position) >= self.p.max_position:
            return

        if obi > self.p.obi_threshold:
            # 买盘强，挂单偏买单
            self.limit_buy(price=mid - spread/2, size=self.p.order_size)
            self.limit_sell(price=mid + spread, size=self.p.order_size)
        elif obi < -self.p.obi_threshold:
            self.limit_buy(price=mid - spread, size=self.p.order_size)
            self.limit_sell(price=mid + spread/2, size=self.p.order_size)
```

## 五、回测引擎：事件驱动 vs 向量化

### 5.1 向量化回测（快但不精确）

```python
def vectorized_backtest(close: pd.Series, signal: pd.Series) -> pd.DataFrame:
    """
    signal: 1=多, -1=空, 0=空仓
    返回每日收益序列
    """
    rets = close.pct_change().fillna(0)
    strat_rets = signal.shift(1) * rets  # t-1 信号 * t 收益
    return strat_rets
```

优点：快（万行数据毫秒级出结果）。缺点：无法模拟滑点、停止单、仓位管理等细节。

### 5.2 事件驱动回测（慢但真实）

事件驱动引擎把市场抽象成事件流：

```python
class EventDrivenBacktester:
    def __init__(self, data: pd.DataFrame, strategy: Strategy, commission_bps=2):
        self.data = data
        self.strategy = strategy
        self.commission_bps = commission_bps
        self.cash = 1_000_000
        self.position = 0
        self.trades = []

    def run(self) -> pd.DataFrame:
        for ts, row in self.data.iterrows():
            event = BarEvent(ts=ts, open=row.open, high=row.high,
                             low=row.low, close=row.close, volume=row.volume)
            self.strategy.on_bar(event)
            self.execute_orders(row)
        return self.equity_curve()

    def execute_orders(self, row):
        for order in self.strategy.pending_orders:
            fill_price = order.price * (1 + self.slippage(row))
            self.cash -= fill_price * order.size
            self.position += order.size
            self.trades.append({...})
```

### 5.3 关键模拟：滑点与延迟

```python
def slippage(row, order_size, adv=row.volume * 50):
    """Almgren-Chriss 线性冲击模型"""
    return 0.1 * (order_size / adv)  # 10bp 基础 + 线性市场冲击

def latency_model(actual_ts, strategy_ts):
    """模拟下单到成交的网络延迟"""
    return actual_ts + pd.Timedelta(milliseconds=np.random.exponential(5))
```

**回测一定要保守**：滑点估高、手续费估高、延迟估高。回测里赚 100 万的策略，实盘可能只剩 30 万。

## 六、绩效评估：不只是 Sharpe Ratio

### 6.1 常用指标

| 指标 | 含义 | 标准 |
|------|------|------|
| Sharpe Ratio | 风险调整收益 | > 1 可考虑，> 2 优秀 |
| Sortino Ratio | 只对下行波动惩罚 | > 2 优秀 |
| Max Drawdown | 历史最大回撤 | < 20% 为佳 |
| Calmar Ratio | 年化收益 / 最大回撤 | > 1 优秀 |
| Win Rate | 胜率 | 因策略而异 |
| Profit Factor | 总盈利 / 总亏损 | > 1.5 为佳 |

```python
def performance_metrics(returns: pd.Series, rf=0.03) -> dict:
    annual_ret = returns.mean() * 252
    annual_vol = returns.std() * np.sqrt(252)
    sharpe = (annual_ret - rf) / annual_vol

    downside = returns[returns < 0]
    sortino = (annual_ret - rf) / (downside.std() * np.sqrt(252))

    equity = (1 + returns).cumprod()
    max_dd = (equity / equity.cummax() - 1).min()

    return {
        'annual_return': annual_ret,
        'annual_vol': annual_vol,
        'sharpe': sharpe,
        'sortino': sortino,
        'max_drawdown': max_dd,
        'calmar': annual_ret / abs(max_dd) if max_dd < 0 else np.nan,
    }
```

### 6.2 过拟合检测

- **样本内 vs 样本外**：保留最后 20% 数据不参与调参
- **参数敏感性**：把策略参数 ±20% 微调，看 Sharpe 是否骤降。骤降说明过拟合
- **walk-forward 分析**：用 2018-2020 训练，2021 测试；再用 2019-2021 训练，2022 测试……以此类推

### 6.3 控制换手率

高频策略看起来 Sharpe 高，但换手率上去了手续费也爆：

```
净 Sharpe = 策略 Sharpe - 换手率 × (佣金 + 滑点)
```

如果换手率 100 倍，每笔 5bp 成本，一年下来就是 5000bp 的"成本墙"。

## 七、执行层：从订单到成交

### 7.1 订单类型

- **Market Order**：市价单，立即成交
- **Limit Order**：限价单，挂单等待
- **Stop Order**：触发价后转市价
- **Iceberg Order**：冰山订单，分批暴露
- **TWAP / VWAP**：算法执行订单

### 7.2 VWAP 算法执行

把大单按历史成交量分布拆小单：

```python
def vwap_schedule(total_size: float, hist_volumes: pd.Series) -> list:
    """返回每 5 分钟应下单的 size"""
    total_vol = hist_volumes.sum()
    fractions = hist_volumes / total_vol
    return [total_size * f for f in fractions]
```

### 7.3 智能订单路由（SOR）

跨多个交易所寻找最优价：

```python
def smart_route(order, order_books):
    """
    order_books: {exchange: book}
    贪婪匹配每个交易所的最优价位
    """
    remaining = order.size
    fills = []
    while remaining > 0:
        best_ex = max(order_books, key=lambda e: order_books[e].best_price(order.side))
        fill_size = min(remaining, order_books[best_ex].best_size(order.side))
        fills.append({'exchange': best_ex, 'size': fill_size, 'price': ...})
        remaining -= fill_size
    return fills
```

## 八、风控：系统的"刹车"

风控分三层：策略风控、组合风控、合规风控。

### 8.1 策略层风控

- 单笔最大仓位（如不超过账户净值 10%）
- 单日最大亏损（如 5% 触发自动停机）
- 单只标的最大持仓时间

### 8.2 组合层风控

```python
class PortfolioRisk:
    def var(self, returns: pd.Series, alpha=0.05) -> float:
        """历史 VaR"""
        return -returns.quantile(alpha)

    def cvar(self, returns: pd.Series, alpha=0.05) -> float:
        """条件 VaR（期望 shortfall）"""
        threshold = returns.quantile(alpha)
        return -returns[returns <= threshold].mean()

    def correlation_risk(self, positions: dict, cov_matrix: pd.DataFrame) -> float:
        """组合波动率"""
        w = np.array([positions[s] for s in cov_matrix.index])
        return np.sqrt(w @ cov_matrix @ w)
```

### 8.3 合规与审计

- 所有订单记录可追溯
- 交易所规定的"防 spoofing"自动检测
- 大额订单（> ADV 5%）触发人工审批

### 9.1 延迟优化

高频交易追求微秒级延迟：

- **语言**：C++ / Rust，避免 GC 停顿
- **网络**：内核旁路（DPDK / Solarflare OpenOnload），跳过 TCP/IP 协议栈
- **硬件**：FPGA 直接做协议解析和信号计算
- **位置**：服务器物理上贴近交易所 matching engine（co-location）

### 9.2 容灾与降级

- 主备双机房，主挂秒级切备
- 网络中断 → 自动平仓
- 数据延迟 → 进入"只读"模式，不下新单

## 十一、经典策略陷阱

### 11.1 幸存者偏差

回测时只用了"现在还存在"的股票，忽略了已退市的。**修复**：用 point-in-time 数据，包含退市股票。

### 11.2 前视偏差（Look-ahead Bias）

策略用了"未来才公布"的数据。**修复**：所有财务数据用报告日期 + 公告日期对齐，而非财年结束日期。

### 11.3 复权处理错误

前复权 vs 后复权混淆，导致信号失真。**修复**：明确策略用哪种复权方式，全程统一。

### 11.4 滑点估计不足

回测假设 1bp 滑点，实盘可能 10bp。**修复**：用 Almgren-Chriss 或更细的微观结构模型估计滑点。

## 结语

量化交易系统的难点不在某个组件，而在于**整条链路的工程一致性**：数据错了策略再好也白搭，回测滑点估计低了实盘亏钱，风控不严一次黑天鹅抹平一年收益。

记住三个原则：

1. **数据质量 > 模型复杂度**
2. **回测保守 > 回测漂亮**
3. **风险控制 > 收益最大化**

把基础设施搭扎实，策略迭代速度才能上来——这才是真正的 alpha 来源。
